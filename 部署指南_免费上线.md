# 免费上线部署指南（足踝矫正智能规划系统）

本指南给出一条**完全免费**、可长期运行的线上部署路径，并把每一步要填的环境变量写清楚。
整套架构由四部分组成，互相通过公网地址连接：

| 组件 | 技术 | 免费平台 | 说明 |
|------|------|----------|------|
| 前端 Web | Next.js | **Vercel** | 原生支持 Next.js，永久免费额度 |
| 主后端 API | NestJS + Prisma | **Render**（Docker，free 计划） | 15 分钟无访问会休眠，首次唤醒 ~30–60s |
| 算法服务 Algo | FastAPI + SAM + VTK | **Hugging Face Docker Space** | 免费 16GB 内存，能跑 Torch/SAM/VTK |
| 数据库 | MySQL | **TiDB Cloud Serverless** | MySQL 兼容，免费额度大，无需改 Prisma 的 mysql provider |

> 为什么这样选：算法服务依赖 Torch + SAM(模型) + VTK，内存/体积大，常见免费容器（512MB）跑不动；
> Hugging Face 免费 Space 提供 16GB 内存，是唯一能免费跑起来的选择。数据库选 TiDB 是因为它 MySQL 兼容，
> 可直接复用本项目现有的 schema 与 5 个迁移，无需迁移到 PostgreSQL。

代码侧已为免费部署做好的改造（无需你再改代码）：
- 算法服务支持 `SAM_MODEL_TYPE=vit_b`（375MB 轻量模型）并在首次预测时**自动下载** checkpoint，不再需要手动放 2.4GB 文件。
- `apps/algo/Dockerfile` 改为 CPU 版 Torch + headless 系统库，可直接作为 HF Docker Space 运行。
- API 新增公开健康检查 `GET /api/health`，供 Render 探活。
- `render.yaml`、`apps/algo/README_SPACE.md`、各 `.env.example` 已就绪。

## 前置：把代码推到 GitHub

Vercel / Render 都从 GitHub 拉代码（HF Space 用它自己的 git 仓库）。

```bash
cd K:/足踝矫正副本
git init
git add .
git commit -m "init: ankle planning system"
# 在 GitHub 新建空仓库后：
git remote add origin https://github.com/<你的用户名>/ankle.git
git branch -M main
git push -u origin main
```
> 仓库已配 `.gitignore` 忽略 `node_modules`、`.next`、本地 db 等。请勿提交 `sam_vit_h_4b8939.pth`（2.4GB），云端会自动下载所需模型。

## 第 1 步：数据库（TiDB Cloud Serverless，MySQL 兼容）

1. 注册 https://tidbcloud.com → 创建一个 **Serverless** 集群（免费）。
2. 进入集群 **Connect**，方式选 **General / Prisma**，得到连接信息（host 形如 `gateway01.xxx.prod.aws.tidbcloud.com`，端口 `4000`，用户名、密码）。
3. 拼成 Prisma 用的 URL（TiDB 需要开启 TLS）：
   ```
   mysql://<用户名>:<密码>@<host>:4000/ankle_app?sslaccept=accept_invalid_certs
   ```
   先在 TiDB 控制台的 SQL 编辑器执行：`CREATE DATABASE IF NOT EXISTS ankle_app;`
4. **建表（跑迁移）**：在本机执行一次即可（用上面的线上 URL）：
   ```bash
   cd apps/api
   # PowerShell:  $env:DATABASE_URL="mysql://...:4000/ankle_app?sslaccept=accept_invalid_certs"
   npx prisma migrate deploy --schema=../../prisma/schema.prisma
   ```
   把这个 URL 记好，后面填到 Render。

> 备选免费 MySQL：Aiven free、Clever Cloud。若你更想用 PostgreSQL（Neon/Supabase），需把
> `prisma/schema.prisma` 的 `provider` 改为 `postgresql` 并重做迁移——本指南默认走 TiDB 以零改动。

---

## 第 2 步：算法服务（Hugging Face Docker Space）

1. https://huggingface.co/new-space → SDK 选 **Docker**（Blank）。
2. 把本仓库 **`apps/algo` 目录下的全部文件**推送到该 Space 的 git 仓库根目录；
   并把 `apps/algo/README_SPACE.md` **改名为 `README.md`**（其 frontmatter 里的 `app_port: 8000` 必须保留）。
   ```bash
   git clone https://huggingface.co/spaces/<用户名>/ankle-algo
   cp -r apps/algo/* ankle-algo/         # Windows 用资源管理器复制也可
   cd ankle-algo && mv README_SPACE.md README.md
   git add . && git commit -m "algo space" && git push
   ```
3. Space 自动构建运行。在 **Settings → Variables** 设：
   - `SAM_MODEL_TYPE = vit_b`
4. 构建完成后地址形如：`https://<用户名>-ankle-algo.hf.space`（记下，填给 Render 的 `ALGO_SERVICE_URL`）。
   - 自检：浏览器打开 `https://<...>.hf.space/health` 应返回 `{"status":"ok"}`。

---

## 第 3 步：主后端 API（Render）

1. https://render.com → New → **Blueprint** → 连接你的 GitHub 仓库（Render 会读取根目录 `render.yaml`）。
2. 它会创建一个名为 `ankle-api` 的 Docker 服务。`JWT_SECRET` 自动生成，其余在 **Environment** 填：
   - `DATABASE_URL` = 第 1 步的 TiDB URL
   - `ALGO_SERVICE_URL` = 第 2 步的 HF Space 地址
   - `CORS_ORIGIN` = 第 4 步的 Vercel 前端地址（先随便填，拿到后再改）
   - `ADMIN_EMAIL` / `ADMIN_PASSWORD` = 你的管理员账号
3. 部署完成后地址形如：`https://ankle-api.onrender.com`。
   - 自检：打开 `https://ankle-api.onrender.com/api/health` 应返回 `{"status":"ok","db":true}`。
   - 容器启动时会自动 `prisma migrate deploy` + seed 管理员（见 `apps/api/Dockerfile` 的 CMD）。

---

## 第 4 步：前端 Web（Vercel）

1. https://vercel.com → New Project → 导入你的 GitHub 仓库。
2. **Root Directory** 选 `apps/web`（关键，monorepo 必须指定）。Framework 会自动识别为 Next.js。
3. **Environment Variables** 添加：
   - `NEXT_PUBLIC_API_URL` = 第 3 步的 Render 地址（不带 `/api`，例如 `https://ankle-api.onrender.com`）
4. Deploy。得到前端地址，例如 `https://ankle-web.vercel.app`。
5. **回到 Render**，把 `CORS_ORIGIN` 改成这个 Vercel 地址并重新部署，跨域才放行。

## 附：本地一键运行（开发自测）

- 一键起三服务（Windows）：双击根目录 `start.bat`（API:3001 / Algo:8000 / Web:3000）。
- 或用 Docker 起全栈：`cd infra && docker compose up -d`（含本地 MySQL）。
- 本地数据库：开发用 MySQL 实例在 `127.0.0.1:3307`（root/1234，库 ankle_app），见 `apps/api/.env`。
- 默认管理员（seed）：`admin@example.com / Admin123!`。

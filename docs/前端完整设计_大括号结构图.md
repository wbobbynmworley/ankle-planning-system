# 微创外固定架足踝畸形矫正系统 — 前端完整设计（大括号结构图）

> 依据《开发计划及验收功能指标》与现有 Next.js 实现整理，便于对照验收与扩展。

---

## 一、前端应用总览（大括号结构）

```
微创外固定架足踝畸形矫正系统 · 前端
{
  公共层
  {
    根布局 (app/layout.tsx)
    { 字体: Inter / Geist；语言: zh-CN；全局样式: globals.css }
    全局样式体系 (globals.css + tailwind.config.ts)
    { 主题色: medical-primary / surface / border / danger / success；组件类: btn-primary, card-medical, input-field, table-medical }
    认证与路由守卫
    { 登录 / 注册 → Token；/dashboard/* → /auth/me 校验 }
  }

  页面与路由
  {
    未认证
    {
      / (首页)           → app/page.tsx
      /login             → app/login/page.tsx
      /register          → app/register/page.tsx
    }

    工作台（已认证）
    {
      /dashboard                    → 工作台首页（最近病例、快捷入口）
      /dashboard/layout.tsx         → 侧边栏导航、用户信息、退出
    }

    病例与规划
    {
      /dashboard/cases              → 病例列表、创建病例（患者信息 + 影像上传）
      /dashboard/cases/new          → 新建病例（若独立页）
      /dashboard/cases/[id]         → 病例详情（附件、规划入口、已有规划列表）
      /dashboard/cases/[id]/plan-2d → 2D 工作台（术前预案 / 术后规划 / 方案调整）
      /dashboard/cases/[id]/plan-3d → 3D 工作台（三维模型 / 矫形路径 / 方案调整）
    }

    系统管理（ADMIN）
    {
      /dashboard/admin              → 系统管理首页
      /dashboard/admin/import       → 批量导入等
    }
  }

  验收功能 ↔ 前端模块映射
  {
    1. X光片数据矫形
    {
      1.1 术前预案
      {
        术前畸形分析     → 病例详情 + 2D/3D 工作台（影像展示、分割、标注）
        术前参数获取     → 2D：正侧位 mm/px、Pose；3D：STL 位姿、参考骨/移动骨
        术前预案生成     → 2D/3D 工作台「生成规划」流程（调用 algo 2D/3D）
        三维转二维映射   → 3D 工作台与 2D 视图联动（若有映射接口）
      }
      1.2 术后规划
      {
        术后矫形分析     → 2D/3D 工作台内目标位姿设置与预览
        术后方案生成     → plan-2d / plan-3d 的规划计算与结果展示
        规划方案输出     → 下载 PDF（病例详情与工作台内）
      }
      1.3 三维模型
      {
        三维模型视图交互与参数调控   → Workbench3D（旋转/平移、位姿滑块、参考系）
        术后三维模型矫形路径演示     → 3D 工作台路径动画 / 步骤预览
        映射二维影像验证分析         → 2D/3D 同病例切换或联动视图（可扩展）
      }
      1.4 方案调整
      {
        足踝矫正时间调整 → 2D/3D 工作台「总天数」或步长相关控件
        足踝矫正读数调整 → 2D：Pose 读数；3D：位姿读数与步长
        足踝矫正换杆调整 → 参数面板或「方案调整」子区（可扩展专用页）
        足踝矫正方案调整 → 工作台内修改目标/参数后重新生成或覆盖方案
      }
    }

    2. 足踝畸形分析工具
    {
      关节角度测量与标准提示  → 2D/3D 工作台内「测量」工具（角度、距离）
      关键点标注             → 2D：关键点/标注层；3D：关键点标注（可扩展）
    }

    3. 电子方案管理
    {
      电子方案保存  → 病例下「保存方案」/ 保存到 Plan 记录（已有）
      电子方案检索  → 病例列表筛选、病例详情「已有规划」列表、全局方案检索（可扩展）
      电子方案维护  → 病例详情内编辑/删除/版本（可扩展）
      电子方案关联  → 方案与病例、与影像附件关联（当前通过 caseId + planId）
    }

    4. 权限管理
    {
      用户角色设置  → 注册/后台：角色 ADMIN / DOCTOR / 患者；前端展示 role
      功能权限设定  → 前端根据 role 显示/隐藏：创建病例、上传、删除、规划、系统管理
      数据权限设定  → API 层控制；前端按角色展示可见病例/方案范围
    }

    5. 参数设置
    {
      足踝畸形参数   → 系统设置页或工作台侧栏（可扩展 /dashboard/settings 或 admin 下）
      固定架位移参数 → 2D/3D 工作台内步长、单位等
      足踝矫正参数   → 同上，与算法参数一致
    }
  }

  前端技术栈与组件
  {
    框架与构建
    { Next.js 14 (App Router)；TypeScript；Tailwind CSS }
    状态与请求
    { 服务端/客户端组件；lib/api.ts 封装 fetch + Token；getCases / getCase / getPlans / uploadFile / triggerPlan 等 }
    2D 工作台
    { components/Workbench2D/ImageCanvas.tsx；正侧位图、分割、Pose、规划触发、PDF 下载 }
    3D 工作台
    { components/Workbench3D/Viewer.tsx；@react-three/fiber + drei；STL 加载、位姿、多骨、规划、碰撞检测 }
    通用 UI
    { 按钮/表单/卡片/表格 类（Tailwind）；lucide-react 图标（可统一引入） }
  }

  建议新增或强化的前端模块（对照验收）
  {
    分析工具增强  { 2D/3D 角度测量 UI、标准参考值提示、关键点标注列表 }
    方案管理增强  { 方案检索页、方案版本/标签、方案关联病例/影像 }
    参数设置页   { /dashboard/settings 或 admin 下：足踝畸形参数、固定架位移参数、矫正参数 }
    方案调整专项  { 时间/读数/换杆 的专用表单或侧栏，与当前工作台内控件统一 }
  }
}
```

## 三、设计规范（与现有实现一致）

- **主题**：医疗感、青绿主色（medical-primary）、白底卡片、清晰层级。
- **布局**：Dashboard 左侧固定导航，主内容区可滚动；工作台为全屏或大块画布 + 侧栏。
- **交互**：主要操作使用 `btn-primary`，次要使用 `btn-secondary`；表格使用 `table-medical`，列表使用 `card-medical`。
- **无障碍**：保持语义化标签与对比度，关键操作可加 `aria-*`。

以上结构图可直接用于验收对照与前端迭代规划；新增「建议新增或强化的前端模块」可按优先级逐步实现。

## 四、纯大括号结构图（精简版）

```
前端应用 {
  公共 {
    根布局 · 全局样式 · 认证守卫
  }
  路由 {
    未认证 { 首页 / 登录 / 注册 }
    工作台 { 工作台首页 · 侧栏(工作台|病例管理|系统管理) }

```

# 流程与数据库协同说明

## 一、业务流程

### 1. 创建病例（医生 / 管理员）

- **谁可操作**：医生、管理员。
- **入口**：病例管理 →「创建病例」。
- **步骤**：选择患者（从用户列表拉取 PATIENT）、填写备注（选填）→ 提交。
- **数据库**：`Case` 表新增一条，`doctorId`=当前用户，`patientId`=所选患者，`status=DRAFT`。

### 2. 上传附件与开始规划

- **谁可操作**：医生、管理员（患者仅可查看）。
- **入口**：病例详情页。
- **上传**：支持正位片(FRONT)、侧位片(SIDE)、STL 三维、其他；类型可由后端根据文件推断或前端指定。
- **开始规划**：
  - 有**正位+侧位** → 显示「开始 2D 规划」→ 调用算法 2D → 写入 `Plan`（PLAN_2D）→ 自动进入 **2D 工作台**。
  - 有 **STL** → 显示「开始 3D 规划」→ 调用算法 3D → 写入 `Plan`（PLAN_3D）→ 自动进入 **3D 工作台**。
  - 同时具备 2D/3D 数据时，两个按钮都显示，由用户选择进入哪个工作台。

### 3. 规划工作台（2D / 3D）

- **入口**：病例详情中点击「开始 2D/3D 规划」后自动跳转，或从「已有规划」中点击「查看」。
- **内容**：展示该病例下对应类型的最新规划结果（总天数、总距离、每日步进等）；规划已自动保存到数据库。
- **操作**：「导出 PDF」→ 调用后端生成规划报告 PDF 并下载。

### 4. 规划完毕：保存与导出 PDF

- **保存**：触发规划时接口已把结果写入 `Plan`（`totalDays`、`totalDistance`、`dailySteps`、`rawPath`），病例状态更新为 `PLANNED`，无需再点「保存」。
- **导出 PDF**：工作台或病例详情中点击「导出 PDF」/「下载 PDF」→ 请求 `GET /plans/:id/pdf` → 后端用当前病例与规划数据生成 PDF 并返回文件。

### 5. 患者端：查看规划报告

- **入口**：患者登录后进入「病例管理」，仅能看到自己作为患者的病例。
- **病例详情**：只读；可看附件列表、已有规划列表。
- **查看报告**：点击某条规划的「查看」进入 2D/3D 工作台查看详情，或直接点击「下载 PDF」获取规划报告。

## 二、数据库协同（增删改查）

| 实体   | 创建 | 查询 | 更新 | 删除 | 说明 |
|--------|------|------|------|------|------|
| **User** | 注册、管理员批量导入 | 列表(按角色)、当前用户 | 管理员可维护 | — | 医生可查 PATIENT 列表用于创建病例 |
| **Case** | 医生/管理员创建(选患者) | 按角色：管理员全部，医生本人，患者本人 | 医生/管理员改状态、备注 | 医生/管理员可删 | 状态：DRAFT → PENDING_PLAN → PLANNED → COMPLETED |
| **File** | 医生/管理员上传(按病例) | 按病例查，权限同 Case | — | 医生/管理员可删 | 类型：FRONT/SIDE/STL/REPORT/OTHER |
| **Plan** | 触发 2D/3D 规划时创建 | 按病例查；单条查(工作台/导出) | 医生/管理员可改步进等 | — | 规划结果存 dailySteps、rawPath |
| **Log** | 关键操作写日志 | 管理员查 | — | — | 审计用 |

- **权限**：病例/文件/规划的访问与修改均按角色与「本人」校验（管理员看全部，医生看本人负责病例，患者看本人病例）。
- **关联**：Case 关联 Patient、Doctor；Case 下挂 File、Plan；Plan 导出 PDF 时只读 Case+Plan，不写新表。

## 三、API 与前端对应

- **病例**：`POST /cases` 创建，`PATCH /cases/:id` 更新，`DELETE /cases/:id` 删除，`GET /cases`、`GET /cases/:id` 列表与详情。
- **文件**：`POST /files/upload/:caseId` 上传，`GET /files/case/:caseId` 列表，`DELETE /files/:fileId` 删除，`GET /files/download/:fileId` 下载。
- **规划**：`POST /plans/trigger` 触发 2D/3D，`GET /plans/case/:caseId` 列表，`GET /plans/:id` 单条，`PATCH /plans/:id` 更新，`GET /plans/:id/pdf` 导出 PDF。
- **用户**：`GET /users?role=PATIENT` 获取患者列表（医生/管理员创建病例时使用）。

以上流程与数据库协同已按「医生/管理员创建病例 → 上传 2D/3D 数据 → 开始规划 → 自动进入对应工作台 → 保存并导出 PDF → 患者查看报告」实现。

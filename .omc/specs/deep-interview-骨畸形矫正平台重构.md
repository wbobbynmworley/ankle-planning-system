## 目标

整理并重构骨畸形矫正平台的**后端流程、页面导航和数据流**，保留核心算法（2D A*、3D A*、VTK碰撞检测、STL转2D）不变的前提下，使整个系统能够：

## 约束

### 医学流程约束

- **2D测量是医学标准**：参照骨畸形矫正专著，2D测量提供"什么是矫正好了"的判断依据
- **3D规划是实现路径**：根据2D测量结果，在三维空间规划矫正路径
- **两者缺一不可**：3D不能替代2D，因为三维空间没有医学标准参照

### 技术约束

- **保留现有算法**：
  - `stl.py` — STL转2D合成X光片
  - `planner_2d.py` — 2D双平面A*路径规划
  - `planner_3d.py` / `planner_3d_ct3d.py` — 3D体素A*规划
  - `collision.py` — VTK精确碰撞检测
- **前端框架**：Next.js 14 + React-Three-Fiber
- **后端框架**：NestJS + Prisma + MySQL
- **算法服务**：Python FastAPI（端口8000）

### 非目标（明确排除）

- 不重新开发核心算法
- 不改变数据库核心模型结构（User, Patient, Case, Plan, Measurement, File）
- 不做UI视觉大改（仅修复导航和流程，不做全新设计）

## 流程规范

### 3D病例流程

```
1. 医生上传患者STL文件
   ↓ 成功后跳转
2. STL转2D → 生成正侧位合成X光片
   ↓
3. 医生在2D图像上手动标记测量点 → 获取术前角度/长度数据
   ↓ 保存测量结果
4. 进入3D工作台 → 选择基准骨 → 选择移动骨 → 碰撞检测 → A*规划
   ↓ 生成每日矫正步数
5. 实时PDF预览（随掩码调整更新）
   ↓ 确认后
6. 导出PDF矫正报告
```

### 2D病例流程

```
1. 医生上传患者X光片（正侧位）
   ↓
2. 直接在X光片上手动标记测量点 → 获取术前角度/长度数据
   ↓ 保存测量结果
3. 进入2D工作台 → SAM分割基准骨/移动骨 → 调整位置 → A*规划
   ↓ 生成每日矫正步数
4. 实时PDF预览（随掩码调整更新）
   ↓ 确认后
5. 导出PDF矫正报告
```

### 页面导航规范

| 当前状态 | 操作后跳转 |
|---------|-----------|
| 上传STL成功 | → STL转2D测量页面 |
| 2D测量保存 | → 3D工作台（或2D工作台） |

| 规划生成完成 | → PDF预览页面 |

### 数据库模型

```
User          — 角色：ADMIN/DOCTOR/PATIENT
Patient       — 关联User，含身份证号
Case          — 关联Patient+Doctor，状态链：DRAFT→PENDING_PLAN→PLANNED→PREOP_DONE→POSTOP_DONE→COMPLETED
Plan          — 2D/3D规划，含totalDays、dailySteps（JSON）
Measurement   — 术前术后测量（角度/长度）
File          — STL/X光片/掩码文件
```

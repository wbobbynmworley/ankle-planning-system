# 本地部署说明（本机运行，所有改动均在此环境）

当前以 **本机本地部署** 为主，访问地址：**http://localhost:3000**。  
界面已按 **专业医疗软件** 风格配置：青绿主色、清晰表格与卡片、侧栏导航、统一表单与按钮样式。

---

## 一、首次准备（执行一次即可）

在**项目根目录**双击或执行：

```bat
setup.bat
```

会完成：创建 `apps/api/.env`、安装 API/Web/Algo 依赖、Prisma 生成与迁移、创建默认管理员。

**前置**：本机已安装 **Node.js 18+**、**Python 3**、**MySQL**，并已使用数据库 **1234**（用户 `root`，密码 `1234`，端口 3306）。若数据库尚未创建，在项目根目录执行：`scripts\create-db-mysql.sql` 或在 MySQL 中执行 `CREATE DATABASE IF NOT EXISTS \`1234\`;`

---

## 二、日常启动（每次开发前）

在项目根目录执行：

```bat
start.bat
```

会依次打开三个窗口：API（3001）、Algo（8000）、Web（3000）。等待约 30 秒后在浏览器打开：

**http://localhost:3000**

默认管理员：**admin@example.com** / **Admin123!**

---

## 三、本机环境变量（已配好可不动）

- **apps/api/.env**  
  - `DATABASE_URL=mysql://root:1234@localhost:3306/1234`  
  - `JWT_SECRET=change-me-in-production`  
  - `ALGO_SERVICE_URL=http://localhost:8000`
- 前端通过 **next.config.js** 的 rewrites 把 `/api` 转到 `localhost:3001`，未设置 `NEXT_PUBLIC_API_URL` 时即用本地 API。

之后所有功能改动、调试均在本地进行，无需再部署到 Railway。

---

## 四、可选：用 Docker 跑全部服务（含 Postgres）

若不想本机单独装 PostgreSQL，可在项目根目录执行：

```bat
cd infra
docker compose up -d --build
```

访问：**http://localhost**（80 端口，Nginx 反向代理到 Web 与 API）。  
停止：`docker compose down`。

---

## 五、端口一览

| 服务   | 端口 | 说明           |
|--------|------|----------------|
| Web    | 3000 | 前端，浏览器访问 |
| API    | 3001 | 后端           |
| Algo   | 8000 | 算法服务       |
| MySQL  | 3306 | 本地数据库 1234 |

总结：**所有改动均用本机本地部署，日常运行 `start.bat`，在浏览器打开 http://localhost:3000 即可。**
